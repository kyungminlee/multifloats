# Build

Compiling and installing after [configuring](configure.md).

```sh
cmake -B build -S .
cmake --build build
```

A default build produces the two installable libraries
(`libmultifloats-lto-<compiler>.a` and `libmultifloatsf-lto-<compiler>.a`;
LTO is on by default). Tests, benchmarks, and comparison harnesses are opt-in
and are **not** pulled into a default build — enable them via the
[configure options](configure.md#cmake-options).

## Codegen runs first

Two artifacts are generated, not hand-written, and the build depends on them:

- **The Fortran source.** `fsrc/multifloats.fypp` is expanded by
  [`fypp`](https://fypp.readthedocs.io/) into `multifloats-quad.f90` /
  `multifloats-noquad.f90` at build time (a `fypp` binary must be on `PATH`
  at configure time). This is transparent — a normal `cmake --build` runs it.
- **The DD constants header.** `src/dd_constants.hh` is generated by
  `codegen/gen_constants.py` (mpmath) and **committed**. A plain build uses
  the committed copy; you only rerun the generator when you change a constant.

Both are covered in [Codegen](codegen.md), including how to regenerate them
and the drift-detection ctests.

## Install

```sh
cmake --install build --prefix /path/to/prefix
```

This installs the archives, the header, the CMake package config
(`find_package(multifloats)` / `find_package(multifloatsf)`), and the
fypp-expanded `.f90` Fortran sources.

## The generated `.mod` note

A Fortran `.mod` file is **compiler- and version-locked** — its binary format
differs across compilers and even releases. So the default distribution model
ships the fypp-expanded `.f90` **source**, and a consumer compiles the module
with their own Fortran compiler. That sidesteps `.mod` incompatibility
entirely.

`-DMULTIFLOATSF_INSTALL_PRECOMPILED_MOD=ON` additionally installs a
compiler-tagged precompiled `.mod` plus a tagged archive, for consumers who
want to skip the module compile and are willing to match the exact compiler.
Prefer the default (source) unless you control both ends of the toolchain.
